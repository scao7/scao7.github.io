Bitnami Wordpress 套件 + AWS ： 一些坑跟一些tips

一些wordpress建站必须知道的事情，不然及其浪费时间

October 9, 2019 Shengting Cao

1 minute read

有用的插件

ELEMENTOR, Polylang, Profile Builder, Really Simple SSL, Simple Local Avatars, WP Mail SMTP，ninja forms

用Gmail SMTP 作为邮件发送的host

在创建一个EC2 实例之后Wordpress 默认的是user@example.com 这个显然不能够发邮件，我的解决方法是用WP Mail SMTP插件来通过Gmail发送邮件。

第一步是要在Google API console 创建一个发送邮件的应用并且记住他的client id和secret key。在插件中setting里面填入就行了。有时候发现在邮件里无法更爱user@example.com这个时候可以改成SMTP来更改邮箱然后再返回Google SMTP 设置。

SSL Certificate

这个永远是一个很麻烦的问题，我第一个网站就是因为SSL的问题崩了，我第一次使用的是Binami 的Wordpress套件。我用AWS certificate manager 创建SSL。 AWS 的SSL一点都不好用，有时候还乱收费。推荐使用letsencrypt 和 Really Simple SSL 插件来强制使用https来访问。

重新制作网站之后我使用letsencrypt 工具配合Really Simple SSL 实现了强制https访问。 简单的使用代码如下：

sudo /opt/bitnami/letsencrypt/scripts/generate-certificate.sh -m YOURMAIL -d YOURDOMAIN

你可以在后面添加多个-d DOMAIN 来进行认证。

使用Really Simple SSL 你可能需要给wp-config.php文档授予权限。

sudo chmod 660 /opt/bitnami/apps/wordpress/htdocs/wp-config.php
sudo chmod 777 /opt/bitnami/apps/wordpress/htdocs/wp-content

给予权限之后来恢复原来的权限:

sudo chmod 640 /opt/bitnami/apps/wordpress/htdocs/wp-config.php
sudo chmod 775 /opt/bitnami/apps/wordpress/htdocs/wp-content

这一套应该是最快速设置SSL认证跟强制https安全访问的方法了。
