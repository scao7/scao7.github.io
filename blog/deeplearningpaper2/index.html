Fully convolutional network for semantic segmentation

Segmentation is different from classification. Instead of predicting just one label classification, segmentation is the trying to predict each class in pixel. The intuitive way to do a segmentation is to run multiple classification network to classify each pixels. There are a lot of redundant calculations if we follow intuitive way. To reduce the calculation time, many researcher are making effort on this topic. The Fully convolutional paper is the first paper to breakthrough this challenge.

Architecture:

  • original classification part

  • Change fully connected layer to fully convolutional layers

  • Generally: if apply a larger image to the network it will produce a heat map instead of single prediction
  • In detail:
  • Upsampling

  • bilinear interpolation

  • use intermediate outputs sum together to get better prediction

Good ideas:

  • Adapting classifiers for dense prediction
  • just change the classification last fully connected layer to fully convolutional layer
  • Shift-and-stitch is filter rarefaction

  • shift and output interlacing can give dense prediction without Upsampling

  • however upsample interpolation is more effective and efficiency

  • Upsampling is backwards strided convolution
  • deconvolution is better then bilinear interpolation
  • Fine-tuning
  • fine-tune all layers by back- propagation through the whole net

U-net

Architecture:

  • 32 convolutional layers

  • (unpadded 3x3 convolution) x2 + (2x2 maxpolling) x 4 + (unpadded 3x3 convolution) x2 + (2x2 deconvolution) * 4 + (unpadded 3x3 convolution) x2 + ReLU activate function

  • double filter number in each downsampling

  • training time of only 10 hours on a NVidia Titan GPU (6 GB)

Good ideas:

  • Data augmentation

  • shift and rotate invariance because of limited data size

  • random elastic deformation

Deep lab

Architecture:

  • Input + deep convolutional network + bilinear interpolation + fully connected CRF(Conditional Random Field)
  • Deep convolutional network: VGG-16 and Resnet-101
  • Atrous convolution reduce the degree of signal downsampling
  • Fully connected CRF to refine segmentation result and better capture boundaries

Good ideas:

  • Atrous convolution
  • Atrous means “hole” in French
  • Just make the filter has more sparsity by adding pad to it
  • The number of parameter remain the same
  • Use Astrous convolution in 2D compare with the traditional convolution
  • Atrous Spatial Pyramid Pooling (ASPP).
  • Pooling with different rate of filters and combine results
  • It is good for recovery information for different scale object (normal but high cost way is sale image and add it to the training sets, ASPP are more efficient)
  • Structured Prediction with Fully-Connected Conditional Random Fields for Accurate Boundary Recovery

  • deeper network has more classification accuracy but with low localization accuracy

  • Traditionally, conditional random fields (CRFs) have been employed to smooth noisy segmentation maps, our goal should be to recover detailed local structure rather than further smooth it

  • Introduce a new energy function and unary potential function check wiki to find more information.
