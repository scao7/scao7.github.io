fundamental papers in classification

  • Krizhevsky, Alex, Ilya Sutskever, and Geoffrey E. Hinton. “Imagenet classification with deep convolutional neural networks.” Advances in neural information processing systems. 2012. pdf (AlexNet, Deep Learning Breakthrough)

  • Simonyan, Karen, and Andrew Zisserman. “Very deep convolutional networks for large-scale image recognition.” arXiv preprint arXiv:1409.1556 (2014). pdf (VGGNet,Neural Networks become very deep!)

  • Szegedy, Christian, et al. “Going deeper with convolutions.” Proceedings of the IEEE Conference on Computer Vision and Pattern Recognition. 2015. pdf (GoogLeNet)

  • He, Kaiming, et al. “Deep residual learning for image recognition.” arXiv preprint arXiv:1512.03385 (2015). pdf (ResNet,Very very deep networks, CVPR best paper)

  • DeepFace: Closing the Gap to Human-Level Performance in Face Verification pdf (Face Recognition)

  • FaceNet: A Unified Embedding for Face Recognition and Clustering pdf (Face Recognition)

Alexnet

Before the deep learning network came out, the normal step to do a image classification following this steps:

  1. Run SIFT (Scale-Invariant Feature Transform) feature detection algorithm
  2. Encode these features into a feature vector
  3. Run an SVM classifier on the feature vector

Architecture of AlexNet:

  • 5 convolutional layers followed by 3 fully-connected layers.
  • 1000-way Softmax applied at the end
  • 60 million total parameters
  • AlexNet was trained on 1.2 million images (ILSVRC-2010) for 5-6 days on two GTX 580 3GB GPUs

Good Ideas:

  • Use RelU(Rectified Linear Unit):
  • networks with ReLUs learn several times faster than equivalent networks with saturating neurons
  • Local Response Normalization
  • Data Augmentation:
  • Using a PCA trick to alter the intensity/color of images to increase size of training set (led to reduction in top-1 error by ~1%)
  • Dropout:
  • Randomly mute half the neurons for each training sample
  • AlexNet applied Dropout to first 2 FC layers
  • “This technique reduces complex co-adaptations of neurons, since a neuron cannot rely on the presence of particular other neurons.”

VGGNet

VGGNet just make AlexNet deeper

Architecture:

  • 8-16 convo layers, 4096 FC, 4096 FC, 1000 FC, (softmax)
  • Fix other parameters of architecture and steadily increase the depth of the network by adding more convolutional layers (using small 3x3 filters w/ stride of 1, and padding to preserve the input dimension, all hidden layers followed by ReLU)

Good Ideas:

  • Network Initialization Strategy
  • Bad initialization can stall learning due to the instability of gradient in deep nets
  • This paper trained the more shallow configuration (A) w/ random initialization and then used these as the first 4 Convo layers and final three FC layers when initializing bigger networks
  • Dense Evaluation (opposed to multi-crop)
  • Dense evaluation entails doing 1 pass of the CNN over an entire scaled image at test time (and its horizontal flip) then doing sum-pooling at the end to get 1 score per channel
  • Key finding: Dense Evaluation and Multi-Crop are complementary!
  • “Scale Jittering
  • During training, they randomly cropped 224x224 patches from rescaled training images (sized anywhere between 256x256 to 512x512)
  • During testing, they passed in different scales of images and sum-pooled the output to arrive at a fixed-size vector of class scores

GoogleNet

Architecture:

  • 22 layers deep, but 12x fewer parameters than AlexNet (8 layers deep) while being much more accurate
  • Inception module
  • started as a case study for an algorithm to build sparse networks
  • Full architecture

Good Ideas

  • Go deeper

  • Auxiliary Classifiers
  • Used auxiliary classifiers connected to intermediate layers to combat the vanishing gradient problem during training

ResNet:

Architecture:

  • Residual Block
  • To solve “The Degradation Problem” - Deeper Networks were more difficult to train!

Good Ideas:

  • Projection shortcut
  • Bottleneck building block
  • The middle layer has smaller input/output dimensions, so we can train faster b/c there are fewer params s
